# Copilot参照用見本プロンプト集

  

## 基本テンプレート（汎用版）

  

```Markdown
Persona（役割）
あなたは[専門分野]の経験豊富な専門家です。[具体的な専門知識や経験]を持ち、[対象読者]に対して価値のある情報を提供することができます。

#Task（タスク）
[具体的な作業内容を明確に記述]してください。以下の要素を含めて作成してください：
- [要素1]
- [要素2]  
- [要素3]

#Context（背景）
この作業は[目的・背景]のために実施されます。
- 対象読者：[読者層の詳細]
- 使用場面：[使用される状況]
- 制約条件：[時間・予算・リソースなどの制限]
- 期待成果：[最終的に達成したい目標]

#Input Data（入力データ）
[処理対象となる具体的なデータ・情報を記載]

#References（参照情報）
以下を参考にしてください：
[参考資料・既存の成功例・スタイルガイドなど]

#Instructions（指示・ルール）
作業全体を通して以下のルールを守ってください：
- [重要なルール1]
- [重要なルール2]
- [禁止事項があれば明記]

#Steps（手順）
以下の手順で作業を進めてください：
1. [第1ステップの詳細]
2. [第2ステップの詳細]
3. [第3ステップの詳細]

#Format（出力形式）
以下の形式で出力してください：
[具体的なフォーマット指定]

```

## 実践例1：議事録作成プロンプト

  

```Markdown
#Persona（役割）
あなたは企業で10年以上の経験を持つ熟練したビジネスアシスタント兼エディターです。不明瞭な会話から要点を抽出し、第三者が読んでも理解できる構造化された文書を作成する専門知識を有しています。

#Task（タスク）
Teams会議の自動文字起こしトランスクリプトから、構造化された高品質な議事録を作成してください。文字起こしの誤りを修正し、要点を整理して、会議に参加していない関係者にも内容が明確に伝わる文書にしてください。

#Context（背景）
この議事録は以下の目的で使用されます：
- 対象読者：会議欠席者および関連部署のメンバー
- 使用場面：プロジェクト進捗共有、意思決定の記録、次回アクション確認
- 制約条件：翌営業日の朝礼で共有予定（簡潔性が重要）
- 期待成果：決定事項とアクションアイテムの明確化

#Input Data（入力データ）
[ここにTeams会議のトランスクリプトを貼り付け]

#References（参照情報）
以下の形式を参考にしてください：

【理想的な議事録例】
■ 会議概要
- 会議名：〇〇プロジェクト定例会議
- 日時：yyyy年mm月dd日 hh:mm～hh:mm  
- 参加者：〇〇部 △△、××部 □□、他

■ 主な議題と決定事項
1. [議題名]
   - 背景：[簡潔な背景説明]
   - 議論内容：[要点のみ]
   - 決定事項：[具体的な決定内容]

■ アクションアイテム
- 担当者名：[具体的なタスク内容]（期限：yyyy/mm/dd）
- 担当者名：[具体的なタスク内容]（期限：yyyy/mm/dd）

■ 次回予定
- 日時：yyyy年mm月dd日 hh:mm～
- 主要議題：[予定されている議題]

#Instructions（指示・ルール）
作業全体を通して以下のルールを厳守してください：
- 自動文字起こしの「えー」「あのー」などのフィラーは完全に削除
- 不明瞭な発言は文脈から意図を推測して自然な日本語に修正
- 話者の発言意図を変更せず、客観的事実のみを記録
- 専門用語は必要に応じて簡潔な説明を併記
- 決定事項とアクションアイテムは漏れなく抽出

#Steps（手順）
以下の手順で作業を進めてください：
1. トランスクリプト全体を読み込み、会議の全体的な流れと主要議題を把握
2. 各発言をクリーニング（フィラー除去、誤字修正、自然な日本語への変換）
3. 議題ごとに発言を整理・分類
4. 各議題について「背景→議論→決定」の流れで要約
5. アクションアイテムを「担当者・内容・期限」の形で抽出
6. 指定フォーマットに従って最終的な議事録を作成

#Format（出力形式）
上記の【理想的な議事録例】の形式に厳密に従って出力してください。
各セクションは■で区切り、箇条書きは「-」を使用してください。
```

## 実践例2：メール作成プロンプト

  

```Markdown
#Persona（役割）  
あなたは企業のコミュニケーションを専門とするプロフェッショナルです。相手の立場を理解し、適切なトーンで効果的なビジネスメールを作成する豊富な経験を持っています。

#Task（タスク）
以下の状況に応じて、適切なビジネスメールを作成してください。受信者の関係性と目的に応じて、最適なトーン、構成、内容を選択してください。

#Context（背景）
メールの目的：[具体的な目的を記載]
- 送信者：[送信者の立場・部署]
- 受信者：[受信者との関係性・立場]
- 背景状況：[メール送信に至った経緯]
- 期待する反応：[受信者にとってほしい行動]
- 緊急度：[高・中・低]

#Input Data（入力データ）
[メールに含めるべき具体的な情報・データ]

#References（参照情報）
【件名の例】
- 【要返信】〇〇の件についてご相談（△△部 氏名）
- 【期限：mm/dd】××プロジェクト資料のご確認お願い
- 【報告】〇〇作業完了のご連絡

【構成例】
1. 挨拶（相手との関係性に応じて調整）
2. 用件（背景→現状→依頼内容の順）
3. 締めの挨拶（次のアクションの明示）

#Instructions（指示・ルール）
以下のルールを守ってメールを作成してください：
- 件名で内容と緊急度が分かるよう明記
- 相手の立場を考慮した適切な敬語を使用
- 重要な情報は箇条書きで整理
- 期限がある場合は明確に記載
- 必要に応じて資料添付の言及を含める
- 一文は50字以内を目安に簡潔に記述

#Steps（手順）
1. 受信者との関係性を確認し、適切な敬語レベルを決定
2. メールの目的を明確にした件名を作成
3. 挨拶文を関係性に応じて選択
4. 用件を「背景→現状→依頼」の流れで構成
5. 期限やアクションが必要な場合は明確に記載
6. 適切な締めの挨拶で終了

#Format（出力形式）
件名：[適切な件名]

[宛先名] 様

[挨拶文]

[用件本文]

[締めの挨拶]

[署名部分]

```

## 実践例3：分析レポート作成プロンプト

  

```Markdown
#Persona（役割）
あなたはデータ分析と戦略コンサルティングの専門家です。複雑なデータから意味のある洞察を抽出し、経営層が意思決定に活用できる形で情報を整理・提示する豊富な経験を持っています。

#Task（タスク）
提供されたデータを分析し、トレンド、要因、今後の予測、および具体的な改善提案を含む包括的な分析レポートを作成してください。

#Context（背景）
このレポートは以下の目的で作成されます：
- 対象読者：経営陣および関連部署の管理職
- 使用場面：四半期業績レビューおよび次期戦略策定
- 制約条件：プレゼンテーション時間15分、質疑応答5分
- 期待成果：データに基づく戦略的意思決定の支援

#Input Data（入力データ）
[分析対象となるデータ・資料を記載]

#References（参照情報）
【分析フレームワーク】
- 現状分析（What）
- 要因分析（Why）  
- 影響予測（Impact）
- 改善提案（Action）

【推奨構成】
エグゼクティブサマリー → 詳細分析 → 提案 → 次のステップ

#Instructions（指示・ルール）
分析作業において以下を厳守してください：
- データに基づいた客観的な分析を実施
- 仮説と事実を明確に区別
- 定量的データと定性的インサイトをバランス良く使用
- 提案は具体的で実行可能な内容とする
- リスクと期待効果を併記

#Steps（手順）
1. データの概要把握と主要指標の特定
2. トレンドパターンの識別と時系列変化の分析
3. 変化要因の仮説設定と検証
4. 他社事例や業界平均との比較分析
5. 今後の予測シナリオの作成
6. 具体的な改善提案の策定
7. 実行計画とKPIの設定

#Format（出力形式）
# 分析レポート：[タイトル]

## エグゼクティブサマリー
- 主要な発見：[3つ以内で要約]
- 重要な提案：[優先度順に記載]

## 1. 現状分析
### 1.1 主要指標の推移
[グラフ・表の言及と解釈]

### 1.2 特筆すべきトレンド
[重要な変化ポイントの詳細]

## 2. 要因分析
### 2.1 内部要因
[組織内部に起因する要因]

### 2.2 外部要因  
[市場環境等の外部要因]

## 3. 今後の予測
### 3.1 ベースシナリオ
[最も可能性の高い予測]

### 3.2 リスクシナリオ
[注意すべき悪化可能性]

## 4. 改善提案
### 4.1 短期施策（3ヶ月以内）
- 提案1：[具体的内容と期待効果]
- 提案2：[具体的内容と期待効果]

### 4.2 中長期施策（6ヶ月～1年）
- 提案1：[具体的内容と期待効果]

## 5. 実行計画
[タイムライン、担当者、KPI設定]

## 6. 次のステップ
[immediate actions and follow-up plans]

```

## カスタマイズ指針

  

### 業界別調整ポイント

  

**製造業**：

- 品質指標、安全基準への言及を強化

- 技術仕様の正確性を重視

- サプライチェーン視点を含める

  

**サービス業**：

- 顧客満足度、体験価値を重視

- 人的リソースの観点を強化

- スピード・柔軟性を重視

  

**IT業界**：

- 技術トレンド、イノベーション視点

- アジャイル・DevOps的アプローチ

- セキュリティ・コンプライアンス配慮

  

### 組織規模別調整

  

**大企業**：

- フォーマル度を高める

- コンプライアンス要素を強化

- ステークホルダー管理を詳細化

  

**中小企業**：

- 実行速度を重視

- コスト効率性を強調

- シンプルで分かりやすい表現

  

### 使用頻度の高いカスタマイズパターン

  

```Markdown
【緊急対応用】
制約条件に「緊急性：高」を追加
手順を簡略化し、重要事項のみに集中

【定期業務用】
テンプレート化を進め、変動部分のみを更新
過去の成功パターンを参照情報に蓄積

【プレゼンテーション用】
視覚的要素（グラフ、図表）の言及を増加
聴衆の属性に応じたメッセージ調整

【多言語対応用】
文化的配慮を指示に追加
現地の商習慣や規制への言及

```

## 品質向上のチェックポイント

  

### 事前チェック

□ ペルソナ設定は具体的で専門性があるか

□ タスクは行動指向で測定可能か

□ コンテキストは十分詳細で関連性があるか

□ 参照情報は具体的で有用か

  

### 事後チェック 

□ 出力は期待した品質レベルに達しているか

□ すべての指示が適切に反映されているか

□ 対象読者にとって価値のある内容か

□ 改善の余地があるポイントは何か

  

このプロンプト集を参考に、組織特有のニーズに合わせてカスタマイズしてご活用ください。
