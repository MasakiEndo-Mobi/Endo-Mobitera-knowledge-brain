# Zettelkastenリンク戦略ガイド

このドキュメントは、Zettelkastenの思想に基づき、ノート間のリンクを最大限に活用するためのベストプラクティスをまとめたものです。

## 1. 基本原則：なぜリンクするのか？

- **文脈の提供**: リンクは、情報が「どのような文脈で」関連しているかを示します。これにより、単なる情報のリストではなく、知識のネットワークを構築します。
- **思考の連鎖**: リンクを辿ることで、アイデアがどのように連なり、発展していくかを可視化し、新たな発見を促します。
- **再利用性の向上**: 個々のノート（知識の原子）をリンクで繋ぐことで、様々な文脈で知識を再利用しやすくなります。

## 2. リンクの種類と実践方法

### 2.1. コンテキスト内リンク（インラインリンク）

文章の流れの中で、ごく自然に関連するノートへのリンクを埋め込みます。

- **方法**: `リンクしたいノート名` を使って、文章中に直接リンクを記述します。
- **例**: 「思考は“外部記憶”に委ねることでワーキングメモリの解放が起こる。」
- **コツ**: リンクのアンカーテキスト（表示されるテキスト）は、リンク先を見なくても文脈が理解できるように工夫します。

### 2.2. 関連ノート・セクション

ノートの末尾に「関連ノート」というセクションを設け、関連するノートへのリンクをリストアップします。

- **例**:
  ```markdown
  ## 関連ノート
  - [[p-24cy-867d|P-24CY_867D]]
  - [[ambient-bar-light|R-技術_アンビエント棒照明]]
  ```

## 3. MOC (Map of Contents) の活用

MOCは、特定のテーマやプロジェクトに関するノートをまとめた「目次」や「ハブ」となるノートです。散らばった知識を構造化し、全体像を把握するために不可欠です。

### MOCの作成タイミング

- ある特定のトピックに関するノートが **5〜10個** 程度たまったら、そのテーマ専用のMOCを作成する良いタイミングです。

### MOCの作り方

1.  **新しいノートを作成**: `MOC_テーマ名.md` のような分かりやすい名前を付けます。（例: `MOC_3Dプリンター技術.md`）
2.  **関連ノートへのリンクをリスト化**: そのテーマに関連するノートへのリンクをリスト形式で記述します。
3.  **構造化と注釈**: 必要に応じて見出しを付けたり、ノート間の関係性について簡単な説明を加えたりして、単なるリンクリスト以上の価値を持たせます。

- **例 (`MOC_Resources_サプライヤ.md`)**:
  ```markdown
  # サプライヤ MOC

  ## 3Dプリンター関連
  - [[markforged-overview|R-サプライヤ_Markforged]]
  - [[umi-overview|R-サプライヤ_UMI]]

  ## 加飾・印刷関連
  - [[hoshiden-fd|R-サプライヤ_ホシデンFD]]
  - [[nakanuma-art-screen|R-サプライヤ_中沼アートスクリーン]]
  ```

## 4. タグとリンクの戦略的な使い分け

タグとリンクは、それぞれ異なる目的を持つツールです。

| 特徴 | タグ (`#tag`) | リンク (`Note`) |
| :--- | :--- | :--- |
| **目的** | **分類とフィルタリング** | **文脈と関係性の提示** |
| **構造** | フラット（横断的） | ネットワーク（構造的） |
| **使い方** | ノートの「属性」を示す（例: `#idea`, `#todo`, `#done`） | ノート間の「思考の流れ」を示す |
| **具体例** | `#24CY` `#DRBFM` | `[[825d-overview|P-24CY_825D]]` は `[[drbfm-overview|R-技術_DRBFM_概論]]` の手法を適用 |

### ベストプラクティス

1.  **タグは状態や大分類に**: プロジェクト名 (`#24CY`)、ステータス (`#active`)、ノートの種類 (`#議事録`) など、後で検索・フィルタリングしたい属性に限定します。
2.  **概念や固有名詞はノートに**: 何度も言及する可能性のあるサプライヤ名、技術名、プロジェクト名は、それ自体を独立したノート（例: `R-サプライヤ_UMI.md`）にし、そこへリンクを張ります。
3.  **MOCへの昇格**: 特定のタグが増えすぎたら、それはMOCを作成する良いサインです。そのタグがついたノートをまとめるMOCを作成し、知識を構造化します。

## 5. 継続的な改善ワークフロー

1.  **デイリーレビュー**: `1_Inbox` のメモを見直し、関連するノートへリンクを追加しながら `Projects`, `Areas`, `Resources` へ移動させます。
2.  **MOCの育成**: 新しいノートを作成したり、既存のノートを編集したりする際に、関連するMOCにもリンクを追加・更新します。
3.  **リンクの追加を習慣に**: 文章を書く際は、常に「この単語や概念は、他のどのノートと繋がるか？」を意識し、積極的に `[[` をタイプします。

このガイドラインに従ってリンクを育てることで、あなたのナレッジベースは単なる情報の倉庫から、アイデアを生み出す強力な「第二の脳」へと進化します。
