# Knowledge Rally — ナレッジベース壁打ちスキル

ナレッジベースを「参照資料」ではなく「共通の思考地図」として使い、
ユーザーとAIが往復しながら理解・発想・検証を深めるラリースキル。

---

## 使うべき場面

- 「〜についてどう考えればいい？」
- 「〜の理解を深めたい」「〜を整理したい」
- 「〜という仮説を試したい」「考えの弱点を見つけたい」
- 「複数のテーマがどうつながるか知りたい」
- 「アイデアを壁打ちしたい」「考えがまとまらない」

## 使うべきでない場面

- 何かを保存したい → `kb-intake`
- 議事録から decision/task 抽出 → `compile-meeting`
- 要件を grill して詰めたい → `requirements-grill`

---

## 実行手順

### Step 0: ラリーの起点を把握する

ユーザーの入力からモードを判断する:

| モード | 説明 | 典型的なユーザーの言葉 |
|---|---|---|
| **explore** | テーマの輪郭を一緒に見つける | 「〜について考えたい」「もやもやしている」 |
| **drill** | 特定論点を掘り下げる | 「〜を深く理解したい」 |
| **connect** | 複数テーマをつなげる | 「〜と〜の関係は？」 |
| **stress** | 仮説の弱点を探す | 「これは本当に正しいか？」 |
| **propose** | 提案・戦略を試す | 「この提案どう思う？」「壁打ちしたい」 |
| **synthesize** | 議論をまとめ整理する | 「まとめて」「整理して」 |

### Step 1: ナレッジベースを読み込む

関連する canonical ドキュメントの **L0 を先に読む**。足りなければ L1 → L2 と深掘りする。

機構設計ドメインでは以下の順で探す:
1. `10_projects/<project>/README.md`（プロジェクト概要）
2. `10_projects/<project>/design-notes/*.md`（過去の設計検討）
3. `10_projects/<project>/test-reports/*.md`（試験結果）
4. `20_components/<category>/*.md`（部品横断知見）
5. `30_standards/*/<file>.md`（関連規格）
6. `50_research/*.md`（技術リサーチ）

### Step 2: ラリー開始宣言

```
**Rally: [テーマ名] — [モード]**

[ナレッジベースから引き出した文脈を2-3文で提示]

[ラリーの最初の一手: 問い・視点・論点のどれか1つだけ]
```

### Step 3: ラリーを続ける

毎ターン以下のルールを守る:

1. **1ターン1スタンス**: 1つのスタンスを選んで実行
2. **短く打つ**: 1レスポンス最大400字（日本語）
3. **必ず1つで終わる**: 最後は問い・反論・観察のどれか**1つ**だけ
4. **ナレッジベース引用**: 主張の根拠はドキュメントのL0/L1から引く。Wikilink `[[doc_id]]` または `[[ファイル名]]` で参照

### Step 4: ラリーの節目を作る

5〜7往復ごと、またはユーザーが「まとめて」と言ったとき:

```
**節目: ここまでで見えてきたこと**

✅ 確認できたこと: [箇条書き 2-3 点]
❓ まだ開いている問い: [箇条書き 1-3 点]
🔗 接続できたテーマ: [接続したKBドキュメントを列挙]

続けますか、それとも別の角度から攻めますか？
```

### Step 5: 着想の記録提案

ラリー中は `kb-intake` を**自発的に提案しない**。ユーザーが「保存したい」「メモして」と明示したときだけ提案する。

### Step 6: リサーチエスカレーション

KBの知識だけでは答えられない問いが浮上したとき、確認付きで提案する:

```
🔎 KB 外調査の候補

ラリーの中で [具体的な論点] について掘り下げたところ、
KBには [何が足りないか] の情報がありません。

どうしますか？
1. WebSearch で調べる
2. メモだけ残して後回し
3. スキップしてラリーを続ける
```

---

## 禁止事項

- 1ターンで複数の問いを投げない
- ナレッジベースを参照せず感覚論だけで応じない
- 結論を急がない
- `kb-intake` をラリー中に自発的に提案しない
- 確認なしで WebSearch を実行しない
